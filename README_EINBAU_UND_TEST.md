# DeutschZ FIX23 - Einbau und Test

## Inhalt

Dieses Paket ist ein kompletter Ersatzstand fuer die aktuell aktiven 5 DeutschZ-Mods:

- DeutschZ_Core
- DeutschZ_ExpansionBridge
- DeutschZ_KotHZ
- DeutschZ_ConvoyZ
- DeutschZ_GroundZero

Enthalten sind ausserdem:

- SteamWorkshop_Beschreibungen mit Links und Changelogs
- AUTO_GENERATED_TEST_CONFIGS_REFERENCE als Referenz fuer automatisch erzeugte Testconfigs

## Wichtig

Der Serverstand davor startete bereits. FIX23 ist deshalb kein kompletter Neubau, sondern ein stabilisierender Startklar-Fix:

- KotHZ Musik aus
- KotHZ Flaggenpfade nachgezogen
- ConvoyZ teststabil mit killbaren Vanilla-Infected
- GroundZero kill-gated statt Stage-Rush
- Marker-/3D-Marker-Intent aktiv
- sichtbare Vanilla-Beacons ueber ExpansionBridge
- Chat-/Popup-Notifications ueber ExpansionBridge
- automatisch generierte Fast-Testconfigs validiert

## Einbau

1. Server stoppen.

2. Die 5 Modordner aus diesem Paket nach folgendem Pfad kopieren und ueberschreiben:

P:\GitHub_DeutschZModZ\DeutschZModZ\mods\

3. Diese Mods neu packen/signieren:

- @DeutschZ_Core
- @DeutschZ_ExpansionBridge
- @DeutschZ_KotHZ(InfectedSiege)
- @DeutschZ_ConvoyZ
- @DeutschZ_GroundZero

4. Workshopbeschreibungen kopieren nach:

P:\SteamWorkshop_Beschreibungen

5. Auf dem Server das DeutschZ-Profil loeschen, damit die neuen Testconfigs automatisch generiert werden:

$profile:DeutschZ/

6. Server starten.

## Test-Erwartung

Nach ca. 60 Sekunden bei mindestens 1 Spieler:

- KotHZ startet ohne Musik.
- KotHZ zeigt Chat/Popup und sichtbaren Marker-Fallback am Eventort.
- KotHZ Flagge sollte nicht weiss bleiben.
- ConvoyZ startet, zeigt Meldungen, spawnt killbare Gegner und laeuft nicht wegen Crashobjekt-Fehler ab.
- GroundZero startet nicht mehrfach/parallel und rauscht nicht durch alle Stages.
- GroundZero verlangt im Test 3 Gegnerkills je Stage.
- Alle Events erzeugen Marker-/3D-Marker-Intent ueber Core/Bridge.

## Marker-Hinweis

FIX23 garantiert sichtbare Vanilla-Fallback-Beacons und Notifications. Es fuegt keine geratenen direkten Expansion-Mapmarker-Calls in Eventmods ein. Falls echte Expansion-Kartenmarker danach noch fehlen, muss der naechste gezielte Schritt in DeutschZ_ExpansionBridge erfolgen, mit verifizierter Expansion Marker API.
